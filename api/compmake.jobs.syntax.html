parsing Module¶

Canonical forms:
[A] except [B] => A minus the elements in B [A] in [B] => intersection of A and B
Rewriting:
not [job_list] => $all except [job_list] except [job_list] => $all except [job_list]

Association:

[A] except [B] except [C] == [A] except ([B] except [C]) [A] in [B] in [C] == [A] in ([B] in [C])
Priority:
in > except > not

compmake.jobs.syntax.parsing.eval_alias(alias)¶

Evaluates the given alias. Returns a list of job_id strings.

The value can have several types: - if it is a string, it is interpreted as a job id - if it is a list, it must be a list of string, interpreted as a job id - if it is callable (FunctionType),

it is called, and it must return a list of strings.

compmake.jobs.syntax.parsing.parse_job_list(tokens)¶

Parses a job list. tokens can be:

  1. a string, in that case it is split()
  2. a list, in which case each element is treated as a token.

NO(If tokens is not empty, then if it evaluates to empty, an error is raised (e.g. “make failed” and no failed jobs will throw an error).)

Returns a list of strings.
