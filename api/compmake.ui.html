commands Module¶

These are the commands available from the CLI.

There are 3 special variables: - ‘args’: list of all command line arguments - ‘job_list’: the remaining argument parsed as a job list. - ‘non_empty_job_list’: same, but error if not specified.

compmake.ui.commands.ask_if_sure_remake(non_empty_job_list)¶

If interactive, ask the user yes or no. Otherwise returns True.

compmake.ui.commands.clean(job_list)¶

Cleans the result of the selected computation (or everything is nothing specified).

compmake.ui.commands.clustmake(job_list)¶

Cluster equivalent of “make”.

Note: you should use the Redis backend to use multiprocessing.

compmake.ui.commands.exit()¶

Exits the shell.

compmake.ui.commands.make(job_list)¶

Makes selected targets; or all targets if none specified.

compmake.ui.commands.make_single(job_list)¶

Makes a single job – not for users, but for slave mode.

compmake.ui.commands.parmake(job_list, n=None)¶

Parallel equivalent of “make”.

Usage:

parmake [n=<num>] [joblist]
compmake.ui.commands.parremake(non_empty_job_list)¶

Parallel equivalent of “remake”.

compmake.ui.commands.remake(non_empty_job_list)¶

Remake the selected targets (equivalent to clean and make).

compmake.ui.ui.interpret_commands(commands_str, separator=';')¶

Interprets what could possibly be a list of commands (separated by ”;”) If one command fails, it returns its retcode, and then the rest are skipped.

Returns 0 on success; else returns either an int or a string describing what went wrong.
